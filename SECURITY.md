# Security Policy

这个项目包含任务执行、工具调用、审批、治理控制面和会话记忆能力。公开协作时，请把安全问题当成高优先级事项处理。

## 报告范围

欢迎报告下面这类问题：

- 权限绕过
- 审批绕过
- 风险策略失效
- tool registry / model route / change request 门禁失效
- 敏感文件读写越权
- 会话记忆泄露
- 审计缺失或可篡改
- 不安全的默认配置或密钥暴露

## 报告方式

在没有专用安全邮箱之前，请不要公开发布可复现利用细节。

建议采用下面流程：

1. 先用私密渠道联系维护者。
2. 提供影响范围、复现步骤、前置条件和潜在后果。
3. 如果有修复建议，可一并提供最小补丁思路。

## 处理原则

- 优先修复可导致越权、执行失控或密钥泄露的问题。
- 在补丁发布前，尽量避免公开完整利用细节。
- 修复后再补充审计、验收脚本和文档说明。

## 本仓库的最低安全要求

- 不提交真实 API Key、密码或访问令牌
- 不提交运行日志、工作产物和本地备份
- 高风险控制面变更优先通过正式 change request 流程
- 任何会影响执行边界的改动都应补最小验收说明
